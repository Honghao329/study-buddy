# 学习伴侣 — 前端设计文档

> 本文档供前端开发者使用，后端 API 已全部就绪，按此文档开发即可对接。
> 推荐技术栈：Taro 4 + React + NutUI + TailwindCSS

---

## 一、产品概述

**学习伴侣**是一款"学习自律 + 伙伴监督"微信小程序。

**核心循环**：打开 → 看今天该做什么 → 签到/打卡 → 伙伴给反馈 → 明天继续

**用户角色**：学生 / 考研党 / 自学者

## 四、全局约定

### 4.1 认证

- 登录后 token 存 Storage，key = `token`
- 用户信息存 Storage，key = `userInfo`
- 所有需要认证的接口在 Header 带 `Authorization: Bearer {token}`
- 接口返回 `code: 401` 时跳转登录页

### 4.2 分页

- 请求参数：`page`（从 1 开始）、`size`（默认 20）
- 返回：`{ list: [], total: number }`

### 4.3 头像

- 后端已统一处理，所有用户相关接口返回的 avatar 字段**永远有值**（无头像的用户返回默认头像 URL）
- 前端直接用 `<Image src={avatar} />` 即可，不需要做空值判断

### 4.4 时间

- 后端返回 `YYYY-MM-DD HH:mm:ss` 格式
- 前端根据需要格式化（如"3分钟前""昨天"等）

### 5.1 首页（今天）

**导航栏**：系统默认，白底黑字，标题"今天"

**页面结构**（从上到下）：

#### A. 签到状态条
```
┌──────────────────────────────────┐
│ 🔥 3天连续 · 累计12天     [签到] │
└──────────────────────────────────┘
```
- 一个白色卡片，左侧 6rpx 橙色竖线（已签到变绿色）
- 左边：emoji + 连续天数（橙色加粗）+ 累计天数
- 右边：未签到 → 蓝色按钮"签到"；已签到 → 绿色"✓ 已签"
- 点击"签到"弹出签到弹窗；点击"✓ 已签"跳转签到日历页

**API**：`GET /api/sign/stats` → `{ streak, totalDays, totalDuration, todaySigned }`

#### B. 今日打卡任务
```
今日打卡                       2/3
┌──────────────────────────────────┐
│ ✅ 考研数学刷题           48人  │
├──────────────────────────────────┤
│ ✅ 读书打卡               20人  │
├──────────────────────────────────┤
│ ○  测试打卡                0人  │
└──────────────────────────────────┘
```
- 标题行：左"今日打卡"，右"done/total"蓝色
- 每个任务一行：checkbox + 任务名 + 参与人数 + 右箭头
- 已完成：绿色填充圆 + 任务名删除线灰色
- 未完成：空心灰圆
- 点击跳转打卡详情页
- 无任务时显示空态 + "去看看"链接

**API**：
- `GET /api/checkin/list?page=1&size=10` → 任务列表
- `GET /api/checkin/today_done_ids` → 今日已完成的任务 ID 数组

#### C. 最新动态
```
最新动态
┌──────────────────────────────────┐
│ 😊 小明完成了签到(60分钟)       │
│ 📖 小红打卡了「读书打卡」       │
│ ...                              │
└──────────────────────────────────┘
```
- 一个白卡片包裹，每条一行：头像 + 描述 + 时间
- 条目间用细分割线

**API**：`GET /api/home/activity` → `[{ avatar, text, time, icon, color }]`

#### D. 热门笔记
```
热门笔记                     更多
┌──────────────────────────────────┐
│ 👤小明                           │
│ 考研数学高分经验                  │
│ 正文预览两行...                   │
│ 👍12  👁36  💬5                  │
└──────────────────────────────────┘
```
- 每篇笔记一张白卡片
- 头像 + 作者 → 标题（粗体2行）→ 正文预览（灰色3行）→ 点赞/浏览/评论数
- 点击跳转笔记详情

**API**：`GET /api/note/public_list?page=1&size=10&sort=hot`

#### E. 签到弹窗
- Bottom Popup
- 字段：学习时长（数字输入）、学习状态（三选一）、学习内容（文本域）
- 提交：`POST /api/sign/do { duration, status, content }`

### 5.2 打卡列表

**导航栏**：标题"打卡"

**页面结构**：
1. 搜索栏
2. 筛选标签：全部 / 我参与的 / 我监督的
3. 任务卡片列表
4. 右下角 FAB 按钮"+"创建任务

**任务卡片**：
- 左侧蓝色竖线
- 标题 + 描述（2行截断）
- 底部：日期范围 + 参与人数
- 已参与/监督中 标签

**API**：
- `GET /api/checkin/list?page=1&size=20&search=xxx`
- `GET /api/checkin/my_joined_ids` — 已参与的 ID
- `POST /api/checkin/create { title, description, start_date, end_date }` — 创建

### 5.3 打卡详情

**页面结构**：

#### A. 头部（蓝色渐变）
- 任务标题 + 描述 + 日期范围
- 角标：你的身份（参与者 / 监督者）

#### B. 数据条（浮起白色卡片）
- 我的打卡 | 总参与 | 浏览量

#### C. 监督关系卡片
- **有监督者时**：头像 + 名字 + 关系描述
  - 你是被监督者："小红正在监督你完成这个任务"
  - 你是监督者："你正在监督这个任务，可以评价和催打卡" + [催打卡]按钮
- **无监督者时**：引导"邀请伙伴监督 → 有人监督，坚持更容易"

#### D. 参与者（头像横滚）

#### E. 打卡记录（列表）
- 每条：头像 + 姓名 + 日期 + 内容
- 有评价时：蓝色左边框卡片显示评分+评语
- 监督者可以对未评价的记录写评价

#### F. 底部固定栏
- 分享按钮 + 打卡主按钮

**API**：
- `GET /api/checkin/detail/:id` → 任务详情 + 监督者 + 参与者
- `GET /api/checkin/records/:id?page=1&size=20` → 打卡记录
- `POST /api/checkin/join { checkinId, content }` — 打卡
- `POST /api/checkin/invite_supervisor { checkinId, supervisorId }` — 邀请监督
- `POST /api/checkin/comment_record { recordId, comment, score }` — 评价
- `POST /api/message/remind { checkinId, targetUserId }` — 催打卡

## 八、注意事项

1. 所有接口返回格式统一：`{ code: 200, data: ..., msg: ... }`
2. `code: 401` 表示未登录或 token 过期，前端应跳转登录页
3. `code: 403` 表示无权限
4. 图片上传用 FormData，字段名 `file`，限制 5MB，仅支持 JPG/PNG/GIF/WebP
5. 头像字段后端已做默认值处理，前端不需要写空值占位逻辑
6. 下拉刷新全部页面都应支持
7. 签到每天只能一次，重复签到会返回错误
